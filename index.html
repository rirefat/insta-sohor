01. How does javascript works?

Ans: JavaScript is a web programming language that manages the dynamic aspects of online sites. JavaScript code is directly executed on a web browser. We add some JavaScript code into a normal Web page. When the browser runs the page, it contains a built-in interpreter that interprets and executes any JavaScript code found on the page. The web browser initially loads a web page, parses the HTML, and generates a Document Object Model (DOM) from the contents. The DOM provides the JavaScript code with a live view of the web page. The DOM initially assembles the HTML and CSS to build the web page. The browser's JavaScript engine then loads JavaScript files and inline code but does not immediately execute the code. It waits for the HTML and CSS to load completely. After that, the JavaScript is executed in the order in which it was written. As a consequence, JavaScript code updates the DOM and the browser renders it.

02. What does javascript event loop do?

Ans: The event loop is the key of asynchronous programming in JavaScript. JavaScript includes an event loop-based runtime architecture that is responsible for running code, collecting and processing events, and performing queued sub-tasks. The event loop is a process that waits for the Call Stack to be empty before attempting to push callbacks from the Task Queue to the Call Stack. When the Stack is cleared, the event loop is triggered, and the Task Queue is checked for available callbacks. If any, it pushes it to the Call Stack, waits for the Call Stack to clear, and then repeats the procedure.
